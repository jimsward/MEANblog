Deploy Your Node App to Heroku

Now that I've deployed a few Nodejs apps to Heroku, I'm seeing a pattern of a sequence of steps to get my code from a repo to Heroku and deploy.

This assumes the goal of a workflow where every push to github will deploy to Heroku.

First, prep the app. We need a Procfile in the root directory. We only have one process - a web process - so, the file consists of one line : 'web: node app'. Or replace the value after the colon with whatever command line you use to start your app. Heroku has some problems with bower_install called from a package.json; I put all the bower_components (angular stuff) on cdns.

  • Clone the app from github
  • cd to the app's newly created repo
  • heroku create appname (creates the app) Heroku assign a name and you will see it below the command line
  • git push heroku master (deploys the app)
  • Launch the app from a browser using the assigned app name
  • Go to the Heoku Dashboard; click the app name; click deploy; go to Connect to Github and choose a repository to connect to for deploys
  • Go down two sections and enable automatic deploys
  • Now, every time you push to this repo on Github the app will automatically deploy.

That's pretty much it.
